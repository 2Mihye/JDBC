싱글톤 패턴 : 객체 지향 디자인 패턴 중 하나로 특정 클래스가 딱 단 하나의 인스턴스만 생성되도록 보장하는 패턴

싱글톤 인스턴스 :
		싱글톤 패턴을 따르는 클래스의 인스턴스를 말하며 오직 한 번만 인스턴화 되고, 어디서든지 이 인스턴스에 접근할 수 있다.
		여러 요청에 동일한 인스턴스를 공유한다.

인스턴스화 :
		실제 객체를 생성하는 과정을 말하고, 생성자를 호출하여 객체를 가져와서 초기화 시킨다.
		객체를 생성하면 해당 객체는 메모리에 할당되고 사용할 수 있게 된다.

DTO(Data Transfer Object) : 제품 정보 저장. get을 활용하여 저장된 정보를 가져옴.
		// DTO는 로직을 가지지 않는 데이터 객체이고, getter, setter 메소드만 가진 클래스를 의미한다.

Model : 데이터베이스와 상호작용하고, 싱글톤패턴을 사용해서 인스턴스가 단 한 번만 생성되도록 보장

View : 사용자로부터 제품 정보를 입력받는 역할

Main : 프로그램 시작점
